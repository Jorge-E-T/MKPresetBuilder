GitHub Setup Guide

1 Create a GitHub Account ▶

GitHub is a free platform for hosting files online. You need an account to host your preset JSON file where the r1 camera can reach it publicly.

Steps

  1. Open your web browser and go to github.com
  2. Click the Sign Up button in the top-right corner
  3. Enter your email address and click Continue
  4. Create a password (at least 15 characters, or 8 characters with a mix of letters, numbers, and symbols)
  5. Choose a username — this becomes part of your preset URL, so pick something clean (e.g. johndoe-presets)
  6. Complete the verification puzzle to prove you're human
  7. Check your email and click the verification link GitHub sends you
  8. When asked about your plan, choose Free (the free plan is everything you need)
💡 Your username appears in every URL for your files. Choose something short and memorable. You can have as many repositories as you want on the free plan.
2 Create a Repository ▶

A repository (repo) is like a folder on GitHub that stores your files. You need one public repo to host your preset JSON file.

Steps

  1. After logging in, click the + icon in the top-right corner of GitHub
  2. Select New repository from the dropdown
  3. Set the Repository name — keep it simple with lowercase letters and hyphens, e.g. r1-presets or camera-presets
  4. Set the Description (optional) — e.g. Custom presets for r1 camera app
  5. ⚠️ Critical: Select Public — your r1 device cannot access private repositories
  6. Check the box for Add a README file (this lets you immediately upload files)
  7. Leave all other settings at their defaults
  8. Click Create repository
⚠️ The repository must be set to Public. If it is Private, the r1 device cannot access it, even if you are logged in on another device.
3 Repository Structure & Folder Setup ▶

Your repository needs a specific structure so the r1 camera can find both your preset JSON file and your preview images.

Required Structure

your-repo-name/
  ├── README.md             (auto-created)
  ├── my-presets.json      (your exported preset file)
  └── public/              (optional: preview images)
      ├── My_First_Preset.png
      └── Another_Preset.png

Preview Image Rules

  • Create a folder named exactly public (all lowercase) at the root of your repository
  • Image filenames must match the preset name exactly, with spaces and special characters replaced by underscores
  • Images must be .png format
  • Example: A preset named Neon City Nights needs an image named Neon_City_Nights.png
  • Preview images are optional — presets without images still work, they just show "no preview available"
💡 You can have multiple JSON files in one repository. Each file can be added as a separate source in the r1 app's Preset File Settings.
4 Upload Your JSON File ▶

Once you have exported your JSON file from this builder, upload it to your GitHub repository.

Uploading Files

  1. Navigate to your repository on GitHub (you should see the README.md file listed)
  2. Click the Add file button (near the top right of the file list)
  3. Select Upload files from the dropdown
  4. Drag and drop your .json file into the upload area, or click choose your files to browse
  5. Scroll down to the Commit changes section
  6. Leave the commit message as-is or type something like Add preset file
  7. Make sure Commit directly to the main branch is selected
  8. Click Commit changes

Creating the public/ Folder (for preview images)

  1. Click Add file → Create new file
  2. In the filename box, type public/ (with the slash) — GitHub will create the folder
  3. Type a placeholder filename like .gitkeep after the slash: public/.gitkeep
  4. Click Commit changes
  5. Now navigate into the public folder and use Add file → Upload files to upload your preview images
💡 To update your presets later, upload a new version of the JSON file with the same filename. GitHub keeps the history but the r1 app always reads the latest version.
5 Get the Raw File URL ▶

GitHub has two different URLs for every file. The r1 camera needs the raw URL — the one that serves the actual file content, not the GitHub webpage.

How to Get the Raw URL

  1. Navigate to your repository and click on your JSON file (e.g. my-presets.json)
  2. You will see the file contents displayed on GitHub's website with a toolbar above it
  3. Click the Raw button in that toolbar (it's on the right side, above the file content)
  4. Your browser will navigate to a page showing just the raw file text
  5. Copy the full URL from your browser's address bar

URL Format

The raw URL always follows this pattern:

https://raw.githubusercontent.com/username/repository-name/main/filename.json

For example, if your GitHub username is johndoe and your repo is named r1-presets with a file called my-presets.json, the raw URL would be:

https://raw.githubusercontent.com/johndoe/r1-presets/main/my-presets.json
⚠️ Never use the regular GitHub page URL (which starts with github.com/ and contains /blob/ or /tree/). Always use the raw URL. The r1 Preset Builder app has a Convert GitHub to Raw button that converts it for you automatically.
6 Connect to Your r1 Camera Program ▶

Once your file is uploaded and you have the raw URL, add it to the r1 camera app so it can load your presets.

Adding the Source in the r1 App

  1. Open the r1 camera program on your device
  2. Tap the Settings button (gear icon)
  3. Scroll down and tap Preset File Settings (located below Import Presets)
  4. In the Add Custom Source section at the bottom, enter a friendly Name for this source (e.g. My Custom Presets)
  5. Type your raw GitHub URL into the URL field
  6. If you typed a regular GitHub URL by mistake, tap the Convert GitHub to Raw button — it will convert it (as best it can) automatically
  7. Tap + Add Source to save it
  8. Go back to Settings and tap Import Presets — your custom presets will now appear in the list alongside the built-in presets

Managing Your Sources

  • You can toggle a source ON or OFF in Preset File Settings at any time without deleting it
  • You can disable the default preset library if you want to use only your own presets (only available when at least one custom source is active)
  • Use the EDIT option to update the URL or name of any saved source
  • Tap DELETE to permanently remove a source
  • Changes take effect the next time you open Import Presets
💡 Custom source presets are always free to import — the credit and unlock system applies only to the built-in preset library.
